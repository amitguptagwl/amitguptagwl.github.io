Run particular

  • Run tests by their position in a feature file
  • Run tests for a story
  • Run tests using a tag expression
  • Run tests for a set of feature files 
$ cytorus run --only "3,4" --spec "path/of/file.feature"
$ cytorus run --skip "3,4" --spec "path/of/file.feature"
$ cytorus run --story "ABCD-1234"
$ cytorus run --tag "@all but not @wip"
$ cytorus run failing

Quick result

  • Quick test: Run tests for a particular page
  • Test further Integration impact: Run tests from a particular page
  • Test integration backward: Run tests up to a particular page
  • Minimal integration tests: Run tests from a particular page to it's next page only
$ cytorus run --story "ABCD-1234" --on "product page"
$ cytorus run --on "product page"
$ cytorus run --from "product page"
$ cytorus run --from "product page" --till 2
$ cytorus run --not-via "product page"
$ cytorus run --via "product page"

Threshold Strategy

Threshold strategy is helpful to keep the CI pipeline green.
  • Fix old failling tests gradually
  • Handle timebound tests 
[{
    tagExpression: "@timebound but not @wip" ,
    min: "100%",
    when: () => (new Date()).getHours() < 17
}]
